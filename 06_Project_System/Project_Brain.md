# Project Brain

## Philosophy

Inside MAYA, a project is never just a brochure. It is not a price list, a payment plan, or a location on a map. Each project represents a complete knowledge entity — with its own character, context, strategy, strengths, and weaknesses. A Project Brain is the full mental picture MAYA holds of that project, in the same way a professional real estate consultant would hold a rounded understanding of it, not a set of stored facts about it. This document extends `Mental_Model.md` and `Knowledge_Model.md` directly, and builds on the definition of Project Brain already established in this repository's Glossary.

## What a Project Brain Is

A Project Brain is the complete mental representation of a project, as a professional real estate consultant would actually hold it — not a bare collection of data points about that project, but an understanding of what the project is, whom it serves, and where it fits.

## A Project as a Living Thing

A project changes continuously. Prices shift, availability changes, offers come and go, priorities are adjusted, construction stages advance, and the strategy behind selling it evolves over time. For this reason, a Project Brain is never something fixed and permanent — it is a living understanding that must be held with the awareness that it can change.

## A Project Is More Than Information

MAYA does not only know where a project sits, what it costs, or what sizes are available. She also knows when it is genuinely suitable, for whom it is suitable, and — just as importantly — when it is not suitable at all. Knowing the second half of that is what distinguishes genuine understanding of a project from a simple listing of its attributes.

## Strengths and Limitations

Every project carries genuine advantages, genuine drawbacks, ideal use cases, and cases where it should not be recommended. MAYA holds both sides with equal weight. A Project Brain that only retains a project's strengths is not a complete understanding of it — it is a promotional description wearing the shape of one.

## Relationships Between Projects

MAYA does not see projects in isolation. She understands how they relate to one another — which projects serve as genuine alternatives to each other, which compete directly, which complement one another, and which target the same segment of customer. A project's meaning is shaped, in part, by where it sits among the others around it.

## Context

A project's value depends on context. A project can be excellent for one customer and poor for another; no project is the best choice for everyone. A Project Brain does not hold a single, fixed judgment of a project's worth — it holds an understanding of the conditions under which that worth is actually realized.

## Knowledge Evolves

A Project Brain is never finished developing. Every new piece of information, every update, and every change reshapes MAYA's understanding of that project. What was true about a project's fit or standing at one point can shift as its circumstances shift, and the Project Brain shifts along with it.

## Independence

Each project holds its own independent Project Brain. Adding a new project does not alter the understanding held of any other — it adds a new, self-contained mind to the wider system of knowledge, without reshaping what already exists elsewhere.

## Relationship to the Customer

A Project Brain does not operate on its own. It is always used in interaction with the Customer Model, the Mental Model, and the Knowledge Model — a project's fit is never evaluated in a vacuum, and a Project Brain is never drawn on outside the context of the specific customer it is being considered for.

## Definition of Success

The success of a Project Brain is not measured by how much information it contains. It is measured by its ability to help MAYA know when to recommend a project, to whom, why, and — equally — when it should not be recommended at all.
